# Best Practices for Web-Based Research

---

## Executive Summary

Effective web-based research is a disciplined, multi-stage process — not simply entering queries into a search engine and accepting the first result. The core practice set is: define the question precisely before searching; search deliberately with intentional query refinement; evaluate sources against consistent quality criteria; verify claims laterally across independent sources; match methods to the kind of question being asked; prefer primary, corroborated, and recent evidence; document uncertainty and provenance; and synthesise findings into explicit, cited conclusions.

The single most consequential improvement any researcher can make is to stop treating the first search engine results page as the complete answer, and to instead triangulate across multiple independent, credible sources before treating a claim as established.

When research outputs are published on the web, best practices extend further: outputs should be accessible, transparent about authorship and method, and written for readers first rather than for search engine ranking.

**The eight most important practical rules:**

1. Start with a narrow, explicitly written question and defined scope.
2. Use search operators and filters intentionally rather than relying on default ranking.
3. Treat search engines as retrieval tools, not truth engines — do not evaluate a source in isolation.
4. Weight sources by evidence quality, recency, expertise, and independence.
5. Match the type of evidence to the kind of claim being made.
6. Separate observed facts, interpretation, and unresolved uncertainty explicitly.
7. Publish or hand off findings with clear sourcing, accessible structure, and transparent authorship.
8. When research involves people or identifiable data, apply consent, privacy, and inclusion practices.

## 3. Key Findings

### 3.1 Define the Research Question Before Searching

The most common failure in web-based research is beginning to search before the question is clearly formulated. An under-specified question produces unfocused queries, which in turn produce a flood of loosely relevant results that are difficult to evaluate or prioritise.

**Best practice:**

1. Write the question out explicitly before opening a browser.
2. Distinguish between the *information need* (what you actually need to know) and the *surface question* (what you might initially type).
3. Identify known constraints: time range, geography, level of evidence required (anecdotal vs empirical vs peer-reviewed), and how the answer will be used.
4. In practice, a short research brief should specify: the decision to inform, the claims to verify, the time range, and the evidence threshold required for confidence.
5. Accept that the question may need to be refined mid-session as you learn the shape of the information space — but start with the best explicit version you can formulate.

A useful heuristic: if you cannot state in one sentence what a *satisfactory* answer would look like, you are not ready to search yet.

### 3.2 Search Strategy and Query Formulation

Empirical usability research consistently shows that most users have poor search skills, and that search success degrades sharply on any attempt beyond the first query [S1, S2].

**Key findings from research:**

- The average web search query is approximately 2 words [S2]. Most users never use advanced search syntax, and when they do, they use it incorrectly [S2].
- First-query success in e-commerce search was 51%; by the third query it had fallen to 18% [S2].
- Users almost never change search strategy — in one study, only 1% of participants adapted their approach after an initial failure [S1]. This rigidity is the primary driver of research failure.
- Users increasingly treat search engines as "answer engines," accepting the first result without evaluating it [S1].

**Best practices for query formulation:**

1. **Start specific, then broaden.** Begin with a precise multi-word query. If it fails, move to a broader term before changing strategy entirely.
2. **Use domain-specific terms.** Identify the vocabulary used by practitioners in the relevant field before constructing queries. Navigation (browsing category pages, wikis, or structured knowledge bases) is often a faster way to learn domain vocabulary than searching blind [S3].
3. **Use advanced operators deliberately.** Site-specific search (`site:example.com`), date-range filters (`before:`, `after:`), phrase-matching quotes, and `filetype:` are valuable when used correctly [S5a]. For topics that imply authoritative domains, search directly within them: `site:.gov`, `site:.edu`, or known standards bodies.
4. **Change strategy, not just wording.** If reformulating the same query yields nothing, change the approach: search from a different angle, seek an authoritative source directly, or use navigation instead of search.
5. **Search for missing counterevidence.** Actively search for sources that contradict your current working hypothesis, not only for those that support it.
6. **Recognise when search is the wrong tool.** For unfamiliar domains, browsing a structured knowledge base (Wikipedia, a library subject guide, a professional association's website) first builds the mental model needed to construct effective queries [S3].

### 3.3 Source Selection and Diversity

No single source is sufficient for any non-trivial research question. The goal is to reach a well-evidenced, triangulated understanding — not to find a single "correct" page.

**Principles:**

1. **Aim for 4–8 independent primary sources** before treating any claim as established.
2. **Diversify source types.** Relying only on web pages found via a single search engine exposes the researcher to algorithmic bias and filter bubbles. Mix: peer-reviewed publications, official documentation, reputable journalism, practitioner sources (with appropriate weight), and primary data sources.
3. **Prioritise by evidence tier:**
   1. Primary documents, standards, official documentation, and original data.
   2. Reputable institutional guidance and peer-reviewed or expert-authored analysis.
   3. Trade press and expert commentary.
   4. Promotional, anonymous, or weakly sourced content.
4. **Trace claims upstream.** Secondary sources (articles reporting on studies) often distort or simplify the original finding. When a claim matters, find and read the primary source it cites.
5. **Use Wikipedia as a map, not a destination.** Wikipedia is a valuable orientation tool and source of references, but should not be cited as a primary source. Its references section often leads directly to high-quality primary sources.
6. **Check for recency.** The appropriate "shelf life" of a source depends on domain: rapidly evolving fields (technology, law, geopolitics) require sources no older than 12–18 months for current claims; foundational or historical topics may use older sources appropriately [S4].

### 3.4 Source Evaluation Frameworks

Several structured frameworks exist for evaluating web sources. The most widely used are CRAAP, SIFT, and the Five Ws. These share a common set of dimensions, complemented by trust and expertise signals emphasised in web publisher guidance.

#### The CRAAP Test (Meriam Library, CSU Chico)

Evaluates sources across five dimensions:

- **Currency** — How recent is the information? Is it current enough for the topic?
- **Relevance** — Does it actually address the research question?
- **Authority** — Who is the author/publisher? What are their credentials?
- **Accuracy** — Is the information supported by evidence? Are sources cited? Can it be verified elsewhere?
- **Purpose** — Why was this content created? To inform, sell, persuade, or entertain?

#### The SIFT Method (Mike Caulfield)

A more action-oriented framework for fast, practical evaluation:

- **Stop** — Pause before sharing or citing. Notice if you are being triggered emotionally (a strong reaction is often a manipulation signal).
- **Investigate the source** — Before reading deeply, check who is behind the source. A few seconds of lateral investigation (see 3.5) is often more efficient than reading the entire page.
- **Find better coverage** — Search for other sources reporting on the same claim or event.
- **Trace claims, quotes, and media to their original context** — Decontextualised quotes and out-of-date statistics are among the most common forms of misinformation.

#### Common evaluation dimensions across frameworks

| Dimension | What to assess |
|-----------|---------------|
| **Authorship** | Named, credentialled author vs anonymous. Institutional affiliation. |
| **Publisher** | Peer-reviewed journal, reputable institution, established media vs personal blog, content farm, PR wire. |
| **Evidence base** | Does the source cite primary data? Are those citations accurate and traceable? |
| **Recency** | Publication date and last-updated date (distinct for many websites). |
| **Purpose and bias** | Commercial interest, advocacy agenda, entertainment focus. |
| **Corroboration** | Is this claim supported by at least two independent sources? |
| **Expertise and experience** | Does the author demonstrate direct expertise or first-hand experience with the topic? [S3a] |
| **Transparency** | Does the page disclose how content was produced, who wrote it, and what its limitations are? [S3a] |

A single weak dimension does not disqualify a source. Two or more weak dimensions — particularly an anonymous author combined with no cited evidence — should raise significant concern.

### 3.5 Lateral Reading and Verification

The most reliable technique for quickly evaluating an unfamiliar source is **lateral reading**: leaving the page and opening new tabs to see what other sources say *about* it, rather than reading the page itself for internal cues [S6].

This approach, studied by researchers observing professional fact-checkers, consistently outperforms **vertical reading** (reading deeply within a single page to assess its credibility). Vertical reading is the default behaviour of most web users and most students — and it is systematically exploited by sophisticated misinformation sites that are designed to appear credible from the inside.

Related to lateral reading is **click restraint**: do not assume the first search result is the best source. Compare multiple results before choosing where to invest reading time [S6a]. Search ranking reflects many factors besides research quality.

**How to apply lateral reading:**

1. Before reading a source in depth, open a new tab and search for the source's name, author, or organisation.
2. Ask three questions: who is behind this information, what is the evidence, and what do other credible sources say? [S6]
3. Look for red flags: prior instances of fabrication, partisan funding, history of errors.
4. Look for green flags: recognition by authoritative institutions, coverage in credible outlets.
5. Trace quotes, numbers, charts, and screenshots back to their original context before treating them as evidence.

**Supporting tools:**

- Wikipedia's articles on news organisations, think tanks, and researchers often provide concise credibility assessments.
- Fact-checking organisations (Snopes, FactCheck.org, Full Fact, AFP Fact Check) maintain records on commonly circulating false claims.
- Web archive services (Wayback Machine) can reveal whether a site has changed its identity or content.

### 3.6 Match Methods to the Kind of Question

Web-based research is stronger when the method fits the question. Not all evidence is interchangeable: qualitative and quantitative sources answer different kinds of questions, and conflating them leads to over-confident or misdirected conclusions.

**Best practice:**

1. **Use qualitative sources for explanation and framing.** Interviews, case studies, and practitioner accounts help answer *why* and *how* questions.
2. **Use quantitative or observational evidence for prevalence and magnitude.** Survey data, usage statistics, and experimental findings help answer *how many* and *how much* questions.
3. **Treat self-reported material carefully.** People describe their beliefs and intentions more reliably than their actual behaviour. Prefer behavioural or observational evidence for claims about what people do [S10, S11].
4. **Avoid overgeneralising from a single method.** A single qualitative source cannot establish prevalence; a single survey cannot explain mechanism.
5. **Prefer mixed evidence when the decision matters.** A finding corroborated across multiple method types carries substantially more weight than one supported only by a single source type.

Many web sources sound definitive while only supporting one narrow kind of claim. Recognising the method behind a source is part of evaluating it.

### 3.7 Reading and Processing Online Content Effectively

Eyetracking research spanning more than two decades establishes that users overwhelmingly *scan* rather than read web content — and this behaviour has remained stable even as web design has changed fundamentally [S5].

**What the evidence shows:**

- People scan web pages in predictable patterns (F-pattern, layer-cake pattern, spotted pattern) and rarely read linearly or completely [S5].
- Motivation, task type, focus level, and personal disposition all moderate how thoroughly a person reads, but even highly motivated users exhibit significant scanning behaviour [S5].
- Pull quotes, inline ads, and unexpected formatting interruptions cause users to abandon linear reading and drop into light scanning [S5].
- On modern search result pages, users exhibit a "pinball pattern" — nonlinear bouncing between results driven by the rich visual layout of search engine results pages [S5].

**Implications for researchers:**

1. **Never assume you have read a page thoroughly** if you have only skimmed it. High-stakes claims warrant deliberate, line-by-line reading of the specific section containing the claim.
2. **Use page structure to navigate efficiently.** Clear headings allow you to locate the relevant section; once there, read carefully.
3. **Do not confuse familiarity with understanding.** Scanning a page creates an illusion of comprehension that does not survive close questioning. If a source will be cited, read the specific passage word for word.
4. **Note publication metadata before investing reading time.** Author, date, and publisher should be checked before reading deeply — not after.

### 3.8 Recognising Misinformation and Misleading Content

Misinformation exists on a spectrum. "Fake news" is a misleading simplification; most misleading online content falls into more nuanced categories that require different detection strategies [S7].

**The 10 categories of misleading content** (EAVI / Beyond Fake News framework):

| Category | Description | Detection approach |
|----------|-------------|-------------------|
| **Propaganda** | Designed to promote a cause, not inform | Check funding and institutional affiliation; lateral read the publisher |
| **Clickbait** | Sensational headline, weak or irrelevant content | Note the gap between headline and body; read the actual article |
| **Sponsored content** | Paid advertising designed to look like editorial | Look for disclosure labels; note lack of critical framing |
| **Satire / Hoax** | Intentionally fictional; sometimes mistaken for real | Lateral read; check Snopes and similar fact-checkers |
| **Error** | Unintentional factual mistakes | Cross-check key claims against primary sources |
| **Partisan** | Accurate but selectively framed to support one side | Seek coverage from opposing perspective; look for cherry-picking |
| **Conspiracy theory** | Unfalsifiable alternative explanations | Look for lack of cited evidence; note resistance to counter-evidence |
| **Pseudoscience** | Claims that mimic scientific language without methodology | Trace to peer-reviewed literature; check for actual study citations |
| **Misinformation** | Inaccurate content spread without intent to deceive | Verify against primary sources |
| **Bogus / fabricated** | Completely invented | Lateral read; use fact-checkers; check original source |

**Practical rules of thumb:**

- A strong emotional reaction (outrage, excitement, fear) is a signal to slow down, not speed up.
- The more extraordinary the claim, the stronger the evidence required before accepting it.
- 80% of middle school students in a Stanford study could not identify sponsored content as distinct from editorial journalism. Adults perform only marginally better. Assume you are susceptible.
- The main goal is not just to spot obvious fabrication but to identify *which category* of misleading content you are dealing with — because each failure mode requires a different check.

### 3.9 Synthesis and Documentation

Gathering sources is not the same as conducting research. The synthesis stage — integrating findings, resolving conflicts, and making explicit the reasoning behind conclusions — is where research value is actually created.

**Best practices:**

1. **Take notes at the source level.** Record the key claim, the source ID, and a direct quotation or precise paraphrase for every finding that may be cited. Do not rely on memory.
2. **Separate fact from interpretation.** Distinguish between what a source says and your inference from it.
3. **Surface conflicts explicitly.** When two sources disagree, document both positions and the nature of the disagreement. Do not silently pick the one that supports your prior view.
4. **Flag single-sourced claims.** Any important claim supported by only one source is unverified and should be marked as such. A practical labelling scheme: *corroborated by independent sources*, *single-sourced but plausible*, *contradicted or unresolved*.
5. **Maintain a source inventory.** A simple table with source ID, URL, date, author, and quality annotation allows the research to be audited and extended later.
6. **Document what was not found.** Noting the absence of evidence on a specific question is itself a finding — it may indicate a genuine gap in the literature or a flaw in the search strategy.

### 3.10 Iterative Research and Knowing When to Stop

Research on the web is not a single pass; it is iterative. Initial findings reshape the question, reveal new leads, and expose the limits of the search strategy.

**Markers of completion:**

1. New sources are adding no new information (saturation).
2. The major claims in the synthesis are corroborated by at least two independent sources.
3. Known conflicts have been documented.
4. The research question has been answered at the level of confidence required for the intended use.

**When to stop and seek expert review:**

1. When the question requires domain expertise to evaluate evidence correctly.
2. When sources conflict and neither is clearly more authoritative.
3. When all available web sources appear to trace back to a single original claim.
4. When the topic involves medical, legal, or financial advice requiring stricter evidence standards.
5. When the consequences of error are high.

### 3.11 Publishing Research Outputs on the Web

When research findings are shared on the web, accessibility and clarity become part of research quality, because they affect whether others can inspect, verify, and reuse the work.

**Publication best practices:**

1. **Visible authorship and dates.** Every published research output should identify who wrote it, when it was published, and when it was last updated [S3a].
2. **Clear sourcing.** Link directly to cited sources where possible. Disclose any sources that could not be fully verified [S3a].
3. **Transparency about method.** Describe how the research was conducted, what was searched, and what was excluded. Uncertainty should be stated explicitly, not smoothed away [S3a].
4. **People-first content.** Write for readers, not for search engine ranking. Content should be directly useful to the intended audience [S3a].
5. **Accessible structure.** Apply WCAG 2.2 fundamentals: clear page title and headings, text alternatives for images, accessible tables, keyboard-usable interactions, readable sequence, and visible focus indicators [S8].
6. **Separate findings from recommendations from raw evidence.** Readers should be able to distinguish what was found, what it means, and what the uncertainty is.

---

### 3.12 Ethics, Consent, and Privacy When Researching People

If web-based research involves human participants, recordings, or identifiable user data, ethical obligations apply in addition to methodological ones.

**Core requirements:**

1. **Informed consent.** Participants must understand what they are consenting to before data is collected [S9].
2. **Participant privacy.** Identifiable data must be handled, stored, and reported in ways that protect individuals [S9].
3. **Accessible participation.** Research recruitment and participation should be usable by people with disabilities and across diverse populations [S9].
4. **Sensitive populations.** Extra care is required when research involves vulnerable groups, sensitive topics, or potentially distressing content [S9].
5. **Research data handling.** Data retention, deletion, and disclosure policies should be established before research begins [S9].

## 5. Conflicts & Open Questions

- **Search success rate figures** [S1, S2] report on different contexts (general web vs e-commerce). The precise figures are domain-specific and should not be treated as universal, though both point in the same direction: rapid degradation after first-query failure.

- **Age of primary sources.** Several sources are from 2001–2014. The behavioural findings (scanning patterns, search strategy rigidity) are likely stable — [S5] explicitly argues this, having replicated 2006 findings in 2019 — but specific metric figures should be treated as indicative rather than current.

- **Lateral reading evidence.** [S6] describes the technique as observed in professional fact-checkers, but the curriculum page does not link to the underlying research paper. The claim that lateral reading outperforms vertical reading is well-supported in the broader media literacy literature but the primary evidence (S12) could not be directly verified in these sessions.

- **Source evaluation weighting conflict.** Google's E-E-A-T framework [S3a] centres trust, expertise, and helpfulness; CRAAP [S4] emphasises currency, relevance, authority, accuracy, and purpose. These are complementary rather than contradictory, but can lead to different weighting decisions for borderline sources.

- **AI-mediated search.** Search platforms increasingly blend classic ranking, AI-generated summaries, and dynamic result presentation. The sources here provide strong operator and credibility guidance but do not fully resolve best practices for AI-mediated search interfaces. This is an open and rapidly evolving area.

- **Misinformation prevalence.** [S7] identifies categories of misleading content but does not provide quantitative prevalence data. This is an open empirical question.

## Addendum: Additional Coverage

This addendum closes two practical gaps identified during audit: the lack of a compact reusable checklist and the need for clearer guidance on AI-assisted search interfaces.

### AI-Assisted Search Verification Rules

1. Treat AI overviews and answer-engine summaries as retrieval aids, not evidence.
2. Open and read the underlying cited sources before using any claim.
3. Verify that the cited source actually supports the summary; generated summaries often compress, overstate, or blend claims.
4. Prefer sources with named authorship, dates, direct documentation, and primary evidence.
5. Run at least one independent non-AI search to check for omitted caveats, counterevidence, or stronger primary sources.
6. Discard summaries that cite no sources, only derivative sources, or sources that do not substantiate the stated claim.
7. For high-stakes claims, require corroboration from at least two independent authoritative or primary sources before treating the answer as established.

### Reusable Web Research Checklist

1. Write the question, scope, time range, and evidence threshold before searching.
2. Start with specific queries, then refine with operators, filters, and authoritative domains.
3. Gather multiple independent sources across source types and trace important claims upstream.
4. Evaluate each source for authorship, evidence, recency, purpose, and corroboration.
5. Lateral read unfamiliar sources before relying on them.
6. Read the exact passage you plan to cite; do not confuse scanning with comprehension.
7. Label important claims as corroborated, single-sourced, or unresolved.
8. Record source ID, URL, date, author, and a note on source quality or limitations.
9. Stop when new sources add little, major claims are corroborated, and remaining uncertainty is explicit.
10. If using AI-mediated search, manually verify every important claim against the underlying sources.
