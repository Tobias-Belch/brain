## Direct Answer

- The best research process is an iterative, evidence-driven backbone, not one rigid method.
- There is no single universal fixed output format for every domain.
- There is a universal most-useful **core template**: standardize the backbone, then customize domain-specific modules.

## Best Research Process (Backbone)

Use this 8-step loop for most research tasks:

1. **Frame the question**
   - Define the decision to support.
   - Write one primary question and clear scope constraints.

2. **Set scope and quality rules**
   - Decide what counts as acceptable evidence.
   - Define inclusion/exclusion and confidence thresholds.

3. **Discover sources broadly**
   - Start wide to avoid tunnel vision.
   - Capture 4-8 credible seed sources.

4. **Narrow and assess quality**
   - Score sources by credibility, recency, method transparency, and corroboration.
   - Prefer primary and standards-based sources where available.

5. **Synthesize by theme**
   - Group findings by question/theme, not by source order.
   - Separate evidence, interpretation, and recommendations.

6. **Surface conflicts and uncertainty**
   - Record contradictions, weak claims, and single-source assertions.

7. **Run a blindspot pass**
   - Check opposing views, recency gaps, geographic bias, adjacent domains, and missing stakeholders.

8. **Report for action**
   - End with decisions, next steps, and what evidence would change conclusions.

## Universal Most-Useful Result Template

```markdown
---
title: <brief title>
---

## Executive Summary

- Direct answer
- Most important findings
- Recommended action
- Confidence / uncertainty note

## Table of Contents

## 1. Research Question and Scope

- Decision being informed
- Primary question and sub-questions
- Scope constraints and exclusions

## 2. Methodology

- Source types consulted
- Search / discovery strategy
- Inclusion / exclusion criteria
- Quality assessment approach
- Method limitations

## 3. Key Findings

### Theme A
### Theme B
### Theme C

[Cite sources inline and surface conflicts directly]

## 4. Source Inventory

| ID | Source | Type | Date | Quality | Contribution |
|----|--------|------|------|---------|--------------|

## 5. Conflicts and Open Questions

- Contradictions between sources
- Provisional claims (single-sourced)
- Unresolved questions

## 6. Blindspot / Gap Analysis

- [ ] Opposing views
- [ ] Recency
- [ ] Practitioner vs. theoretical balance
- [ ] Geographic / cultural variation
- [ ] Adjacent domains
- [ ] Negative results
- [ ] Stakeholder perspectives

## 7. Recommendations and Next Steps

1. Immediate action
2. Follow-up research
3. Decision checkpoint: what would change this conclusion
```

## What To Customize vs Keep

Customize:

- Add comparison matrices for market/tool decisions.
- Add participant profiles/tasks for UX research.
- Add eligibility/risk-of-bias tables for systematic reviews.
- Add implementation implications for technical research.

Rarely remove:

- Methodology
- Source Inventory
- Conflicts and Open Questions
- Blindspot / Gap Analysis

These four sections preserve auditability and prevent false certainty.

## Practical Rule

- Use one stable template across the team.
- Scale depth by stakes and complexity.
- Adapt internals with modules instead of creating a new template each time.
